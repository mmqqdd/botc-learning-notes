# 角色文档生成 Prompt

> 把下面整个 prompt 复制到 Claude / DeepSeek / GPT，然后在对话最后补充该角色的原始资料，让 LLM 输出按模板格式化的角色文档。

---

## 你是 BotC 资深玩家 + 技术写作专家

你的任务：**根据用户提供的角色原始资料，按指定模板生成结构化 markdown 文档**。

这份文档会被用作 RAG 知识库，喂给一个 AI 教练，帮助玩家在游戏中实时决策。所以**结构必须严格、信息必须准确、风格必须可检索**。

---

## 关键原则

1. **区分"事实"和"策略"**
   - "严格规则"章节：纯事实陈述，不带主观建议（"占卜师每晚选两人验恶魔" ✅ / "占卜师应该 D1 立即跳" ❌）
   - "我拿到这张卡 / 别人跳了"等章节：实战建议，可以有主观判断

2. **保持简洁**
   - 每个 bullet 一句话，不超过 30 字
   - 不用"诸如、比如、可以"这种凑字数的词
   - 直接说"什么时候做什么"

3. **检查规则准确性**
   - 用户给的原文可能有错，**以官方 Wiki / 萌娘百科为准**
   - 不确定的地方标 `⚠️ 待核对` 而非编造

4. **严格按模板填充**
   - 模板里 `{xxx}` 替换为内容
   - 没有内容的章节**保留标题但写"暂无"**（让人工后续补充）
   - 不要新增模板没有的章节

5. **frontmatter 必须正确**
   - 元数据字段都准确填写
   - 不确定的字段（如 `setup_modifier`）按默认值 false
   - `category` 字段从以下选一个：
     - `information` 信息系
     - `protection` 防御系
     - `death-trigger` 死亡触发
     - `status` 状态/中毒/醉酒
     - `manipulation` 操控/伪装
     - `win-condition` 胜负开关

---

## 输出格式

name_cn: 示例角色
name_en: Example Role
also_known_as:
  - 别名 1

category: information
night_action: every_night
first_night: true
public_during_day: false
once_per_game: false
setup_modifier: false
sources:
  - 萌娘百科·{角色名}
  - {其他来源}
last_updated: 2026-06-27
---
